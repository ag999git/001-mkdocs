





### Python Conceptual Questions -- With Answers and Examples

#### Conceptual Questions

------------------------------------------------------------------------

##### a. What does "assignment" mean in Python? What is the assignment operator?

-   **Assignment** means storing a value inside a variable.
-   Python uses the **assignment operator** `=` to bind a value to a
    variable.
-   The variable simply becomes a *name* pointing to an object in
    memory.

Example:

``` python
x = 10     # assigns value 10 to variable x
name = "Alice"
```

------------------------------------------------------------------------

##### b. Why is `=` an assignment operator, not a mathematical "equals to" symbol?

-   In mathematics, `=` means **both sides are equal**.
-   In Python, `=` means **store the value on the right into the
    variable on the left**.
-   It does *not* compare values --- it *creates a binding*.

Example:

``` python
x = 5     # means "store 5 in x"
```

To compare values, Python uses `==`.

------------------------------------------------------------------------

##### c. Explain membership operators `in` and `not in`.

-   They check if a value exists inside a *collection* (`string, list`,
    `tuple, set, dict`).
-   Return **True** or **False**.

Example:

``` python
'a' in 'cat'          # True
5 in [1, 2, 3, 5]     # True
'a' not in ['x','y']  # True
```

For dictionaries: - `in` checks **keys**, not values.

Example:

``` python
'apple' in {'a': 'apple'}     # False
'a' in {'a': 'apple'}         # True
```

------------------------------------------------------------------------

##### d. What is the difference between `==` and `is`?

-   `==` checks **value equality** → Are the contents same?
-   `is` checks **identity** → Do both variables point to the *same
    object*?

Example:

``` python
a = [1, 2, 3]
b = [1, 2, 3]
c = a

a == b   # True  (same values)
a is b   # False (different objects)
a is c   # True  (same object)
```

------------------------------------------------------------------------

##### e. Why can you "assign to a variable" but not "assign to a literal"?

-   A variable is a *name* that can store a value.
-   A literal (like `5`, `"hello"`, `[1,2]`) is a fixed value and
    **cannot** be overwritten.

Invalid:

``` python
5 = x     #  Error
```

Valid:

``` python
x = 5     #  variable can store 5
```

------------------------------------------------------------------------

##### f. What is floor division? What is its operator?

-   Floor division returns the **largest integer less than or equal to
    the result**.
-   Operator: `//`.

Example:

``` python
7 // 2    # 3
7 // -2   # -4  (floors toward negative infinity)
```

------------------------------------------------------------------------

##### g. If dividend is positive but divisor is negative, is remainder positive or negative?

-   In Python, **the remainder has the same sign as the divisor**.
-   So if divisor is **negative**, remainder is **negative**.

Example:

``` python
7 % -3   # -2
```

------------------------------------------------------------------------

##### h. List the six comparison operators in Python.

  Operator   Meaning
  ---------- --------------------------
  `==`       Equal to
  `!=`       Not equal to
  `>`        Greater than
  `<`        Less than
  `>=`       Greater than or equal to
  `<=`       Less than or equal to

------------------------------------------------------------------------

##### i. What are identity and membership operators?

-   **Identity operators** (`is`, `is not`)\
    Check if two variables refer to the **same object**.

-   **Membership operators** (`in`, `not in`)\
    Check if a value exists inside a **collection**.

------------------------------------------------------------------------

##### j. Why is it important to understand object identity in Python?

-   Because Python variables store **references** to objects, not the
    objects themselves.
-   Helps avoid bugs when modifying **mutable** data types (lists,
    dicts).

Example:

``` python
a = [1, 2]
b = a
b.append(3)
print(a)   # [1, 2, 3]  (both refer to same list)
```

------------------------------------------------------------------------

##### k. Why must you be careful using `%` on negative numbers?

-   Because in Python,\
    **a % b always has the sign of b (the divisor)**.

Example:

``` python
-7 % 3    # 2
7 % -3    # -2
```

This differs from languages like C, where the sign follows the
**dividend**.
